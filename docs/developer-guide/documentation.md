# ドキュメントのルール

## ファイルの配置場所

| ドキュメントの種類 | 配置場所 |
|------------------|----------|
| Claude（AI）向けのルール | `.claude/rules/` |
| 人間向けのルール（日本語） | `docs/developer-guide/` |

両方のディレクトリを常に同期させてください。ルールを追加・変更した場合は、両方のファイルを更新してください。

## 同期のルール

- `.claude/rules/` にあるルールファイルには、必ず `docs/developer-guide/` に対応する日本語版を用意する
- ルールを別ファイルに切り出した場合は、親ファイルにも参照を記載する（例: `[commit-message.md](./commit-message.md) を参照してください`）

## 使用言語

- `.claude/rules/` — 英語のみ
- `docs/developer-guide/` — 日本語のみ

## 書き方のスタイル

- 簡潔・正確に書く（チュートリアル形式にしない）
- 構造化されたルールには表を使う（例: ファイル名と用途の対応）
- 順序のある手順には番号付きリストを使う
- 順序のないルールには箇条書きを使う
